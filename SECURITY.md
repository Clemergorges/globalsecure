# Security Policy

## Supported Versions
Only the latest version of GlobalSecureSend is actively supported.

## Reporting a Vulnerability
If you discover a security vulnerability, please report it responsibly:

- Do not open public GitHub issues for vulnerabilities.
- Provide steps to reproduce, logs, and impact assessment.
- Contact the project maintainer privately.

## Disclosure Policy
We follow responsible disclosure. Vulnerabilities will be acknowledged and patched before public disclosure.

## Security Best Practices for Contributors
- Never commit secrets or API keys.
- Use environment variables for sensitive data.
- Follow OWASP guidelines.
- Run tests before submitting PRs.
